Model 3: Rep Counter

AttributeDetails
PurposeCount repetitions of detected exercise
ArchitectureNOT a neural network — signal processing algorithm
InputTime series of joint angles (e.g., knee angle for squats)
Algorithm1. Track primary joint angle over time
2. Apply smoothing (exponential moving average)
3. Peak/valley detection with hysteresis thresholds
4. Count transitions: extended → contracted → extended = 1 rep
Latency< 1ms (pure math)
DeploymentPure TypeScript on mobile
// Simplified rep counter logic

class RepCounter {


  private angleHistory: number[] = [];


  private state: 'up' | 'down' = 'up';


  private repCount = 0;


  private readonly upThreshold: number;    // e.g., 160° for squat standing


  private readonly downThreshold: number;  // e.g., 90° for squat bottom



  addAngle(angle: number): { reps: number; phase: string } {


    this.angleHistory.push(angle);


    const smoothed = this.ema(angle);

    

    if (this.state === 'up' && smoothed < this.downThreshold) {


      this.state = 'down';


    } else if (this.state === 'down' && smoothed > this.upThreshold) {


      this.state = 'up';


      this.repCount++;


    }


    return { reps: this.repCount, phase: this.state };


  }


}

        Input: (T, 17, 2) array — T frames of 17 keypoints
    

        Output: feature vector for classifier
    

        """
    

        features = []
    

        for frame_kps in keypoints_sequence:
    

            # 1. Joint angles (8 key angles)
    

            angles = [
    

                calc_angle(frame_kps[5], frame_kps[7], frame_kps[9]),   # left elbow
    

                calc_angle(frame_kps[6], frame_kps[8], frame_kps[10]),  # right elbow
    

                calc_angle(frame_kps[11], frame_kps[13], frame_kps[15]),# left knee
    

                calc_angle(frame_kps[12], frame_kps[14], frame_kps[16]),# right knee
    

                calc_angle(frame_kps[5], frame_kps[11], frame_kps[13]), # left hip
    

                calc_angle(frame_kps[6], frame_kps[12], frame_kps[14]), # right hip
    

                calc_angle(frame_kps[7], frame_kps[5], frame_kps[11]),  # left shoulder
    

                calc_angle(frame_kps[8], frame_kps[6], frame_kps[12]),  # right shoulder
    

            ]
    
        

            # 2. Normalize keypoints (hip-centered, scale-invariant)
    

            hip_center = (frame_kps[11] + frame_kps[12]) / 2
    

            torso_len = np.linalg.norm(frame_kps[5] - frame_kps[11])  # shoulder-hip
    

            normalized = (frame_kps - hip_center) / (torso_len + 1e-6)
    
        

            # 3. Velocities (frame-to-frame displacement)
    

            # computed across sequence
    
        

            features.append(np.concatenate([angles, normalized.flatten()]))
    
    

        return np.array(features)  # shape: (T, 42)
    


    def calc_angle(a, b, c):
    

        """Angle at point b formed by points a-b-c, in degrees."""
    

        ba = a - b
    

        bc = c - b
    

        cosine = np.dot(ba, bc) / (np.linalg.norm(ba) * np.linalg.norm(bc) + 1e-6)
    

        return np.degrees(np.arccos(np.clip(cosine, -1.0, 1.0)))

                nn.Linear(64, 1),
    

                nn.Sigmoid()  # 0-1 form score

        def forward(self, x):

    12.3 — Scaling Failures to Watch For

    📈 Growth Phase Problems:


    1-100 users:    Nothing breaks. You think you're fine.
    

    100-1K users:   MongoDB queries slow down (add indexes).
    

    1K-10K users:   LLM server overloaded (add queue + rate limiting).
    

    10K-50K users:  Video processing backlog (add worker pool).
    

    50K+ users:     Need GPU autoscaling, CDN, database sharding.
